#### 1. **Welcome Screen** (`index_enhanced.html`)
- **Purpose**: Collect user personalization
- **Inputs**: Avatar image, voice sample
- **Outputs**: Uploaded files, session start
- **State**: File selection status

#### 2. **Video Call Interface** (`static/app.js`)
- **Purpose**: Real-time communication
- **Inputs**: Camera, microphone, user speech
- **Outputs**: Video frames, audio transcripts
- **State**: Call status, mic status, connection

#### 3. **Chat Panel** (`static/app.js`)
- **Purpose**: Display conversation history
- **Inputs**: User messages, AI responses
- **Outputs**: Visual message display
- **State**: Message list, scroll position

### Backend Components

#### 1. **Flask Application** (`app.py`)
- **Purpose**: Web server and API
- **Responsibilities**:
  - Serve HTML/CSS/JS
  - Handle file uploads
  - Manage WebSocket connections
  - Route API requests

#### 2. **SocketIO Handler** (`app.py`)
- **Purpose**: Real-time bidirectional communication
- **Events**:
  - `connect`: Client connection
  - `disconnect`: Client disconnection
  - `video_frame`: Video processing
  - `user_message`: Conversation handling

#### 3. **AI Engine** (`app.py`)
- **Purpose**: Generate intelligent responses
- **Components**:
  - Emotional context system
  - Conversation history manager
  - Gemini API integration
  - Vision analysis (optional)

#### 4. **Voice System** (`app.py`)
- **Purpose**: Speech synthesis
- **Components**:
  - ElevenLabs integration (cloned voice)
  - gTTS fallback (default voice)
  - Audio encoding (base64)

#### 5. **File Manager** (`app.py`)
- **Purpose**: Handle user uploads
- **Responsibilities**:
  - Validate file types
  - Secure file storage
  - Generate unique filenames
  - Manage upload folder

## Performance Optimization

### Current Optimizations
1. **Video Frame Throttling**: 2 seconds between frames
2. **Audio Queueing**: Sequential playback prevents overlap
3. **Lazy Loading**: Components load as needed
4. **Async Operations**: Non-blocking file uploads
5. **Canvas Compression**: JPEG quality 0.5

### Future Optimizations
1. **CDN**: Serve static files from CDN
2. **Caching**: Redis for session data
3. **WebSocket Pooling**: Connection reuse
4. **Image Optimization**: Compress uploads
5. **Database**: Persistent storage

---

This architecture provides a solid foundation for an emotionally intelligent AI assistant while maintaining flexibility for future enhancements!
